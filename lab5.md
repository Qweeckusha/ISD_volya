# Лабораторная работа №5 – Функциональные требования
## Информационная система представительства туристической фирмы в зарубежной стране

---

## Введение

Данный документ описывает функциональные требования к информационной системе представительства туристической фирмы. Требования сгруппированы по логическим модулям, соответствуют бизнес-процессам предметной области и напрямую покрывают перечень из 15 типовых запросов, указанных в техническом задании.

---

## Функциональные требования по модулям

### Модуль: Туристы и учет контингента (FR-TUR)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-TUR-01 | Регистрация данных туриста | Внесение ФИО, паспортных данных, пола, возраста, наличия детей и предпочтений по размещению на основе заявок из РФ |
| FR-TUR-02 | Категоризация туристов | Автоматическое распределение по категориям («отдых», «груз», «дети») с проверкой правила: дети не могут перемещаться без родителей |
| FR-TUR-03 | Ведение истории посещений | Сохранение данных о всех поездках: даты, отели, экскурсии, сданный груз для формирования полной карточки туриста (соответствует Запросу 4) |
| FR-TUR-04 | Поиск туриста | Быстрый поиск по ФИО или номеру паспорта с выводом всех связанных данных и истории операций |

### Модуль: Визовое оформление (FR-VIS)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-VIS-01 | Формирование визового пакета | Автоматическая генерация комплекта документов для отдела эмиграции на основе карточки туриста |
| FR-VIS-02 | Учет статуса визы | Отслеживание и изменение статуса (подано, получено, отказ) с возможностью добавления комментариев |
| FR-VIS-03 | Формирование таможенных списков | Генерация списков туристов для таможни в целом и по указанным категориям (соответствует Запросу 1) |

### Модуль: Размещение и гостиницы (FR-HTL)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-HTL-01 | Бронирование номеров | Резервирование мест в партнерских гостиницах с учетом предпочтений туристов и свободного фонда |
| FR-HTL-02 | Формирование списков расселения | Создание отчетов по размещению в разрезе гостиниц и категорий туристов (соответствует Запросам 2, 5) |
| FR-HTL-03 | Учет загрузки гостиниц | Фиксация количества занятых номеров и проживающих лиц за заданный период |

### Модуль: Экскурсионное обслуживание (FR-EXC)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-EXC-01 | Ведение каталога экскурсий | Хранение расписания, данных о принимающих агентствах, стоимости и лимитах мест |
| FR-EXC-02 | Запись на экскурсии | Регистрация туристов на выбранные экскурсии с автоматической проверкой доступности мест |
| FR-EXC-03 | Формирование списков для агентств | Генерация отчетов «турист — экскурсия — агентство» для передачи партнерам |
| FR-EXC-04 | Аналитика экскурсий | Выявление самых популярных маршрутов и ранжирование агентств по качеству услуг (соответствует Запросу 7) |

### Модуль: Груз и складской учет (FR-CRG)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-CRG-01 | Ведение весовых ведомостей | Фиксация данных по грузу «shop-туристов»: маркировка, взвешивание, параметры упаковки |
| FR-CRG-02 | Формирование отгрузочных документов | Генерация ведомости отправки с указанием количества мест, веса, стоимости упаковки и страховки, итоговой суммы |
| FR-CRG-03 | Статистика грузооборота | Учет веса, количества мест и типов воздушных судов, вывозивших груз (соответствует Запросам 9, 12) |

### Модуль: Рейсы и авиационный учет (FR-FLT)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-FLT-01 | Учет рейсов | Ведение базы данных рейсов: дата, тип воздушного судна, пассажирская и грузовая вместимость |
| FR-FLT-02 | Расчет загрузки рейса | Автоматический подсчет количества пассажиров, веса груза и объемного веса для указанной даты (соответствует Запросу 8) |
| FR-FLT-03 | Сводка по рейсу | Формирование полного набора данных по группе рейса: состав туристов, гостиницы, груз, маркировка, бирки (соответствует Запросу 15) |

### Модуль: Финансы и отчетность (FR-FIN)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-FIN-01 | Учет финансовых операций | Внесение статей доходов и расходов: размещение, перевозки, экскурсии, визы, аэропортовые сборы, непредвиденные траты (соответствует Запросу 11) |
| FR-FIN-02 | Генерация финансового отчета | Формирование полного отчета для головной фирмы по группе или категории туристов (соответствует Запросу 10) |
| FR-FIN-03 | Расчет рентабельности | Автоматическое вычисление соотношения доходов и расходов представительства (соответствует Запросу 13) |
| FR-FIN-04 | Анализ структуры потока | Расчет процентного соотношения отдыхающих туристов и участников shop-туров в целом и за указанный период (соответствует Запросу 14) |

### Модуль: Аналитика и общие запросы (FR-ANL)
| Идентификатор | Требование | Обоснование / Связь с запросом |
|--------------|-----------|-------------------------------|
| FR-ANL-01 | Подсчет посещаемости | Получение количества туристов, побывавших в стране за период, с разбивкой по категориям (соответствует Запросу 3) |
| FR-ANL-02 | Статистика по экскурсиям | Подсчет общего числа туристов, воспользовавшихся услугами экскурсионных агентств за период (соответствует Запросу 6) |
| FR-ANL-03 | Экспорт данных | Выгрузка сформированных списков и отчетов в форматы PDF, XLSX, CSV для печати или передачи внешним организациям |

---

## Заключение

Сформированный перечень функциональных требований полностью покрывает бизнес-процессы представительства туристической фирмы: от приема заявок и визового оформления до организации размещения, экскурсий, грузовой логистики и формирования финансовой отчетности. 
> Унифицированная кодировка идентификаторов `FR-[Модуль]-[Номер]` обеспечивает прямую трассировку требований к задачам предметной области и упрощает дальнейшее тестирование и разработку системы.
